# 校圈小店 AI 增长助理 PRD

**版本**：V1.0  
**产品阶段**：MVP / 两周试用验证版  
**目标用户**：学校周边商家老板 / 店长  
**产品定位**：面向学校周边小微商家的轻量 AI 运营工具，帮助商家结合天气、校历、校园节点和店铺信息，自动生成今日促销方案、社群文案、朋友圈文案、小红书内容、海报文案和评论回复话术。

---

## 1. 产品背景

学校周边的小微商家，如奶茶店、咖啡店、快餐店、小吃店、打印店、美甲店、理发店等，通常依赖学生客流，但缺少专业运营能力。商家老板或店长日常主要精力放在出餐、服务、备货和收银上，很难持续进行内容营销、活动策划和私域维护。

同时，校园消费具有明显的时间节点特征。例如开学季、考试周、毕业季、社团招新、运动会、雨天、降温、放假前后等，都会影响学生消费需求。若商家能够及时结合校园节点推出合适的活动，并通过微信群、朋友圈、小红书、抖音等渠道发布内容，就有机会提升到店客流和复购率。

因此，本产品希望通过 AI 降低小微商家的运营门槛，让商家只需填写基础信息，即可一键生成适合当天校园场景的营销内容。

---

## 2. 产品目标

### 2.1 业务目标

1. 帮助学校周边商家降低内容营销和活动策划成本。
2. 提高商家在校园节点下的活动响应速度。
3. 通过两周免费试用验证商家使用频率、内容发布意愿和付费可能性。
4. 为后续收费版、小程序版或 SaaS 版本积累真实使用数据。

### 2.2 用户目标

1. 商家可以快速完成注册、登录和店铺信息配置。
2. 商家可以获取当前城市天气和附近学校校历。
3. 系统可以自动识别开学季、考试周、放假、雨天、降温等校园经营节点。
4. 商家可以一键生成今日促销方案、微信群文案、朋友圈文案、小红书标题与正文、海报文案和评论回复话术。
5. 商家可以记录活动发布时间、优惠券核销数、到店人数和销售额变化，用于简单复盘。

---

## 3. 用户画像

### 3.1 核心用户

| 用户类型 | 用户特征 | 核心需求 |
|---|---|---|
| 学校周边餐饮商家老板 | 奶茶、咖啡、快餐、小吃店老板，人员少，运营能力弱 | 想提升学生到店量，但不会持续做内容 |
| 店长 / 店员 | 负责日常收银、出餐、发朋友圈和微信群消息 | 希望快速生成可直接复制发布的文案 |
| 校园生活服务商家 | 打印店、美甲店、理发店、自习室等 | 希望结合校园节点做精准活动 |

### 3.2 使用场景

1. 早上开店前，商家查看今日天气和校园节点，生成当天活动方案。
2. 中午或下午客流低谷期，商家生成微信群促销文案吸引学生到店。
3. 考试周、毕业季、开学季前，商家生成主题活动和海报文案。
4. 商家收到学生评论、咨询或差评时，使用评论回复助手生成回复话术。
5. 活动结束后，商家记录核销数、到店人数和销售额，进行简单复盘。

1. 暂不接入真实支付系统。
2. 暂不提供复杂 CRM 会员系统。
3. 暂不自动发布到小红书、抖音、朋友圈等第三方平台。
4. 暂不提供多人协作权限管理。
5. 暂不提供完整 BI 数据看板。
6. 暂不承诺自动抓取所有学校校历，优先支持手动录入、上传文件或配置链接。

```text

    ↓
```

## 6. 功能需求

## 6.1 账号注册与登录

### 功能说明

系统支持商家注册账号和登录账号。MVP 阶段默认账号和密码均为 `admin`，方便本地演示和测试。

### 功能需求

| 编号 | 需求描述 | 优先级 |
|---|---|---|
| A-01 | 用户可以通过账号和密码登录系统 | P0 |
| A-02 | 默认账号为 admin，默认密码为 admin | P0 |
| A-03 | 用户登录失败时，系统提示账号或密码错误 | P0 |
| A-04 | 登录成功后进入首页或首次信息填写页 | P0 |
| A-05 | 系统保存登录状态，短时间内无需重复登录 | P1 |

### 验收标准

1. 输入 `admin / admin` 可以成功登录。
2. 输入错误账号或密码时，不能进入系统。
3. 首次登录用户进入店铺基础信息填写页。

---

## 6.2 首次登录基础信息填写

### 功能说明

用户第一次登录后，需要填写店铺基础信息。该信息将作为后续 AI 生成活动方案和文案的基础上下文。

### 信息字段

| 字段 | 类型 | 是否必填 | 示例 |
|---|---|---|---|
| 店铺名称 | 文本 | 是 | 东门柠檬茶 |
| 主营业务 | 单选 / 文本 | 是 | 奶茶 / 咖啡 / 快餐 / 打印 / 美甲 |
| 主要商品或服务 | 文本 | 是 | 柠檬茶、芋泥奶茶、咖啡 |
| 商品 / 服务价格 | 文本 | 是 | 12-25 元 |
| 所在城市 | 文本 | 是 | 北京 |
| 店铺地址 | 文本 | 否 | 学校东门 50 米 |
| 附近学校 | 文本 | 是 | 北京外国语大学 |
| 目标学生群体 | 多选 | 否 | 本科生、研究生、留学生、考研学生 |
| 可接受优惠方式 | 多选 | 否 | 第二杯半价、满减、套餐折扣、会员券 |
| 营业时间 | 文本 | 否 | 10:00-22:00 |
| 发布渠道 | 多选 | 否 | 微信群、朋友圈、小红书、抖音 |

### 功能需求

| 编号 | 需求描述 | 优先级 |
|---|---|---|
| B-01 | 用户首次登录时必须填写店铺名称、主营业务、价格、城市、附近学校 | P0 |
| B-02 | 用户提交后，系统保存店铺信息 | P0 |
| B-03 | 用户后续可以在设置页修改店铺信息 | P1 |
| B-04 | AI 生成内容时自动读取店铺信息作为上下文 | P0 |

### 验收标准

1. 用户未填写必填项时不能提交。
2. 填写完成后可以进入首页。
3. 再次登录时不重复强制填写基础信息。
4. 修改店铺信息后，AI 输出内容应同步变化。

## 6.4 天气获取

### 功能说明

系统根据店铺所在城市获取当前天气，用于判断雨天、降温、高温、寒潮等可能影响学生消费的因素。

### 天气字段

| 字段 | 示例 |
|---|---|
| 城市 | 北京 |
| 日期 | 2026-06-24 |
| 天气 | 小雨 |
| 当前温度 | 22℃ |
| 最高 / 最低温 | 27℃ / 19℃ |
| 降水概率 | 70% |
| 风力 | 3 级 |
| 天气标签 | 雨天、降温 |

### 功能需求

| 编号 | 需求描述 | 优先级 |
|---|---|---|
| D-01 | 系统根据城市调用天气 API 获取天气信息 | P0 |
| D-02 | 系统将天气转化为经营标签，如雨天、降温、高温 | P0 |
| D-03 | AI 生成活动时自动加入天气因素 | P0 |
| D-04 | 天气接口失败时，允许用户手动输入天气 | P1 |

### 验收标准

1. 系统可以展示当前城市天气。
2. 雨天时，AI 可以生成适合外卖、热饮、到店避雨等场景的促销方案。
3. 降温时，AI 可以优先推荐热饮、暖食等内容。

| 是否开学季 | 否 |
| 是否毕业季 | 否 |

| 今日经营建议时段 | 12:00-13:30，19:00-22:00 |

| 编号 | 需求描述 | 优先级 |
|---|---|---|
| E-01 | 用户可以配置附近学校名称 | P0 |
| E-02 | 用户可以手动录入校历信息 | P0 |
| E-03 | 用户可以上传校历文件 | P1 |
| E-04 | 用户可以配置校历链接 | P1 |
| E-05 | 系统调用 AI API 分析校历并提取关键日期 | P0 |
| E-06 | 系统识别考试周、放假、开学季、毕业季等标签 | P0 |
| E-07 | 系统根据上下课时间推荐活动发布时间 | P1 |

### 验收标准

1. 用户输入学校校历后，系统能够提取考试周、放假时间等信息。
2. 如果当前日期处于考试周，首页展示“考试周”标签。
3. 生成内容时能够结合考试周生成“复习续命”“图书馆夜间套餐”等方案。

## 6.6 AI API 接口设置

### 功能说明

用户可以在设置页配置 AI API 接口。系统通过该接口完成校历分析、活动生成、内容生成和评论回复。

### 配置字段

| 字段 | 类型 | 是否必填 | 示例 |
|---|---|---|---|
| API Base URL | 文本 | 是 | https://api.openai.com/v1 |
| API Key | 密码文本 | 是 | sk-xxxx |
| 模型名称 | 文本 | 是 | gpt-4o-mini / deepseek-chat |
| Temperature | 数字 | 否 | 0.7 |
| 最大输出长度 | 数字 | 否 | 2000 |
| 是否启用流式输出 | 开关 | 否 | 是 |

### 功能需求

| 编号 | 需求描述 | 优先级 |
|---|---|---|
| F-01 | 用户可以配置 AI API Base URL | P0 |
| F-02 | 用户可以配置 API Key | P0 |
| F-03 | 用户可以配置模型名称 | P0 |
| F-04 | 系统提供测试连接按钮 | P0 |
| F-05 | API Key 在前端展示时需要脱敏 | P0 |
| F-06 | 调用失败时展示错误原因 | P1 |

### 验收标准

1. 用户填写 API 信息后可以保存。
2. 点击测试连接后，系统能返回连接成功或失败。
3. API Key 不应明文展示在页面上。

## 6.7 校园节点识别

### 功能说明

系统根据当前日期、天气、校历、店铺信息，自动识别当前适合营销的校园节点。

### 节点类型

| 节点类型 | 触发条件 | 示例营销方向 |
|---|---|---|
| 开学季 | 当前日期接近开学时间 | 新生套餐、开学第一杯 |
| 考试周 | 当前日期处于考试周 | 图书馆续命、夜间热饮、复习套餐 |
| 毕业季 | 当前日期接近毕业典礼 | 合照套餐、毕业纪念、聚餐优惠 |
| 社团招新 | 用户手动配置或校历识别 | 社团团购、多人套餐 |
| 雨天 | 天气为雨、雪或高降水概率 | 外卖优惠、热饮、避雨套餐 |
| 降温 | 今日温度明显下降 | 热饮、暖食、夜宵 |
| 运动会 | 校历识别运动会 | 能量补给、团体套餐 |
| 周末 | 当前日期为周六 / 周日 | 休闲、聚会、约会场景 |

### 功能需求

| 编号 | 需求描述 | 优先级 |
|---|---|---|
| G-01 | 系统自动生成今日校园节点标签 | P0 |
| G-02 | 支持用户手动新增节点 | P1 |
| G-03 | 支持用户删除不准确的节点 | P1 |
| G-04 | 生成活动方案时优先使用今日节点 | P0 |

### 验收标准

1. 雨天时首页出现“雨天”标签。
2. 考试周时首页出现“考试周”标签。
3. 用户可以手动补充“社团招新”“运动会”等节点。

## 6.8 今日活动生成

### 功能说明

系统基于店铺信息、天气、校园节点和用户设定的优惠方式，生成适合当天的活动方案。

### 输出内容

| 输出项 | 示例 |
|---|---|
| 活动名称 | 图书馆续命奶茶夜 |
| 活动目标 | 提升 19:00 后到店量 |
| 活动时间 | 19:00-22:00 |
| 活动机制 | 凭学生证第二杯半价 |
| 推荐商品 | 芋泥奶茶、热柠檬茶 |
| 适合渠道 | 微信群、朋友圈、门店海报 |
| 注意事项 | 优惠力度不要超过毛利范围 |

### 功能需求

| 编号 | 需求描述 | 优先级 |
|---|---|---|
| H-01 | 用户点击“生成今日活动”后，系统输出完整活动方案 | P0 |
| H-02 | 活动方案应结合天气、校历和店铺主营业务 | P0 |
| H-03 | 用户可以修改活动目标和优惠力度后重新生成 | P1 |
| H-04 | 用户可以保存满意的活动方案 | P1 |

### 验收标准

1. 奶茶店在考试周生成的方案应围绕复习、晚自习、图书馆等场景。
2. 快餐店在中午时段生成的方案应围绕午餐高峰、套餐、快速出餐等场景。
3. 雨天生成的方案应体现外卖、避雨、热饮等要素。

## 6.9 内容一键生成

### 功能说明

系统根据今日活动方案，一键生成适配不同渠道的内容。

## 7. AI Prompt 脚本设计

### 7.1 今日活动生成 Prompt

```text
你是一个熟悉校园商圈的小微商家运营顾问。
请根据以下信息，为商家生成今日促销方案。

【店铺信息】
店铺名称：{store_name}
主营业务：{business_type}
主要商品/服务：{products}
价格范围：{price_range}
附近学校：{school_name}
目标学生群体：{target_students}
营业时间：{business_hours}
可接受优惠方式：{discount_options}

【外部信息】
当前日期：{date}
当前天气：{weather}
温度：{temperature}
天气标签：{weather_tags}
校历信息：{calendar_summary}
校园节点：{campus_tags}
推荐经营时段：{recommended_time_slots}

请输出：
1. 今日活动名称
2. 活动目标
3. 活动时间
4. 活动机制
5. 推荐商品/服务
6. 适合发布渠道
7. 风险提示

要求：
- 活动要适合学校周边学生消费场景。
- 优惠力度不能过度夸张。
- 文案要具体、可执行。
- 不要编造不存在的商品或学校活动。
```

### 7.2 多渠道内容生成 Prompt

```text
你是一个校园本地生活内容运营助手。
请根据以下活动方案，生成适合不同渠道发布的内容。

【活动方案】
{campaign_plan}

【店铺信息】
{store_profile}

请生成：
1. 微信群文案：50-100 字，直接、有行动引导。
2. 朋友圈文案：100-150 字，自然、有生活感。
3. 小红书标题：3 个备选。
4. 小红书正文：150-300 字，包含体验感和校园场景。
5. 小红书标签：5-8 个。
6. 抖音短视频脚本：15-30 秒，包含镜头、画面、口播。
7. 海报文案：标题、副标题、活动规则、行动引导。

要求：
- 不要使用虚假夸张表达。
- 不要承诺无法保证的效果。
- 语言适合学生群体。
- 内容要便于商家直接复制使用。
```

## 13. 风险与应对

| 风险 | 表现 | 应对方式 |
|---|---|---|
| 商家不会配置 API | 不知道 API Key 是什么 | MVP 阶段可由运营人员代配置，后续提供默认模型服务 |
| 校历获取不稳定 | 不同学校校历格式不同 | 支持手动录入和文件上传，先解决核心学校 |
| AI 内容质量不稳定 | 生成内容空泛或不适合店铺 | 固化 Prompt 模板，加入店铺信息和历史活动数据 |
| 商家不愿意付费 | 觉得只是文案工具 | 强调活动核销、到店人数和复购效果，而不是单纯生成文案 |
| 内容发布链路断裂 | 生成后商家不发布 | 提供一键复制和渠道发布建议，降低操作成本 |

## 14. 后续迭代方向

1. 增加商家行业模板，如奶茶店、咖啡店、打印店、美甲店、理发店等。
2. 增加节日和校园热点自动识别能力。
3. 接入优惠券核销二维码，形成完整活动闭环。
4. 增加活动效果分析，看不同渠道、不同时间段的转化效果。
5. 增加小程序版本，降低商家使用门槛。
6. 增加服务商后台，支持校园代理帮助多个商家运营。
7. 后续可设置基础版、增长版和代运营版收费模式，但 MVP 阶段暂不收费。

## 15. 一句话总结

校圈小店 AI 增长助理不是一个泛化聊天机器人，而是一个围绕“校园节点 + 店铺信息 + 天气校历 + AI 内容生成 + 活动复盘”的小微商家运营工具。MVP 阶段重点验证商家是否愿意使用 AI 生成营销内容，以及这些内容是否能带来真实发布、到店和复购行为。
